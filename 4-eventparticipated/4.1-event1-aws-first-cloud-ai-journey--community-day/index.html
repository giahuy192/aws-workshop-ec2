AWS First Cloud AI Journey – Community Day

Event Objectives

  • Share the latest trends in Cloud Computing and Generative AI on the AWS platform.
  • Introduce modern cloud architectures and best practices for AI application development.
  • Demonstrate how to build intelligent applications using Amazon Bedrock, AWS Lambda, API Gateway, and other AWS services.
  • Explore Large Language Models (LLMs), AI Agents, and Multi-Agent Systems through practical examples.
  • Provide opportunities for networking and knowledge sharing within the AWS and AI community.

Speakers

  • Vy Lam – Senior Business Systems Analyst, VPBank
  • Thảo Nguyễn – GenAI Engineer, VIB
  • Mai Nguyễn – GenAI Engineer, VIB
  • Uyên Lê – GenAI Engineer, VIB
  • Anh Phạm – Cloud Consultant, GoAsiaPacific Vietnam
  • Thịnh Nguyễn – DevOps Engineer, FCAJ
  • Tịnh Trương – Platform Engineer, GoTymeX
  • Đức Đào – Solutions Architect, Cloud Kinetics

Key Highlights

Context Is Everything – Making AI Actually Work for You

The workshop emphasized that context is one of the most important factors affecting the quality of AI-generated responses.

Key topics included:

  • Why AI produces inaccurate answers when contextual information is insufficient.
  • The concept of Second AI Brain for retaining and utilizing knowledge.
  • Combining Prompt Engineering with contextual information to improve response quality.
  • Designing AI systems capable of remembering and reasoning based on previous interactions.

Cloud Architecture with Amazon Bedrock

A complete serverless AI architecture was introduced using AWS services.

Main services included:

  • Amazon CloudFront
  • Amazon S3
  • Amazon API Gateway
  • AWS Lambda
  • Amazon Bedrock
  • Amazon DynamoDB
  • AWS SAM
  • React Application

The architecture demonstrated how users send requests through CloudFront and API Gateway, Lambda processes business logic, Amazon Bedrock performs AI inference, while S3 and DynamoDB store application data.

Benefits include:

  • Fully serverless deployment
  • Automatic scalability
  • Reduced infrastructure management
  • Lower operational costs
  • Faster AI application development

Research on Large Language Models (LLMs)

The speakers introduced the research paper:

“Non-Determinism of Deterministic LLM Settings”

Main insights:

  • The same prompt can still generate different responses.
  • Various hidden factors influence LLM consistency.
  • Understanding model behavior helps developers produce more reliable AI systems.

Understanding Temperature, Top-P and Top-K

The workshop explained how these parameters affect AI output.

  • Temperature controls creativity and randomness.
  • Top-P limits candidate words based on cumulative probability.
  • Top-K limits the number of candidate tokens available for prediction.

Understanding these parameters enables developers to optimize AI responses for different business scenarios.

Single-Agent vs Multi-Agent Architecture

One of the most interesting sessions compared traditional Single-Agent systems with Multi-Agent Architecture.

Limitations of Single-Agent systems:

  • Limited context
  • Single point of failure
  • Poor scalability
  • Limited reasoning capability

The Virtual Credit Committee architecture demonstrated multiple specialized AI agents working together:

  • Manager
  • Financial Analyst
  • Market Analyst
  • Team Evaluator
  • Risk Assessor
  • Compliance Agent

This collaborative approach significantly improves reasoning quality, accuracy, maintainability, and scalability.

Key Takeaways

Cloud Computing

  • Learned how modern cloud architectures simplify AI deployment.
  • Understood how serverless services reduce infrastructure management.
  • Gained practical knowledge of AWS cloud-native application design.

Artificial Intelligence

  • Context management is essential for high-quality AI systems.
  • Prompt Engineering alone is insufficient without proper contextual information.
  • LLM behavior can be optimized through parameter tuning.

Modern Architecture

  • Amazon Bedrock enables rapid deployment of Generative AI applications.
  • Multi-Agent Architecture is more scalable and reliable than traditional Single-Agent solutions.
  • Serverless Architecture improves flexibility while reducing operational costs.

Applying to Work

Knowledge gained from this workshop can be applied in many practical projects.

Possible applications include:

  • Developing AI chatbots using Amazon Bedrock.
  • Designing Serverless Architecture with AWS Lambda and API Gateway.
  • Building intelligent knowledge assistants using contextual memory.
  • Applying Multi-Agent Architecture to enterprise AI systems.
  • Optimizing Prompt Engineering techniques.
  • Deploying scalable AI applications on AWS Cloud.

Event Experience

Attending the AWS First Cloud AI Journey – Community Day was an excellent opportunity to learn from experienced professionals working in Cloud Computing and Artificial Intelligence.

Learning from experts

Industry experts shared practical experiences in deploying AI solutions on AWS. Their presentations provided valuable insights into designing scalable cloud-native architectures and integrating Generative AI into enterprise applications.

Hands-on technical knowledge

I gained a deeper understanding of Amazon Bedrock, AWS Lambda, API Gateway, CloudFront, S3, and DynamoDB by following the complete architecture demonstrated during the workshop.

The explanation of Context Engineering helped me understand why AI systems require more than just good prompts to produce accurate responses.

Exploring modern AI technologies

The sessions about Large Language Models, Temperature, Top-P, Top-K, and Multi-Agent Systems significantly improved my understanding of modern AI technologies.

The comparison between Single-Agent and Multi-Agent Architecture clearly demonstrated how multiple specialized AI agents collaborate to solve complex business problems more effectively.

Networking opportunities

The event also provided valuable opportunities to communicate with AWS professionals, AI engineers, cloud consultants, and other participants who share similar interests in Cloud and Artificial Intelligence.

These discussions expanded my technical perspective and introduced me to many real-world industry practices.

Lessons learned

  • Context is one of the most critical components of modern AI applications.
  • Amazon Bedrock greatly simplifies Generative AI deployment.
  • Serverless Architecture reduces infrastructure complexity while improving scalability.
  • Multi-Agent Systems represent an important future direction for enterprise AI.
  • Combining Cloud Computing with Generative AI creates powerful and flexible intelligent applications.
