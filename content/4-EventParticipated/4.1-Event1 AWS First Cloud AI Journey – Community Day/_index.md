### Key Highlights

#### Context Is Everything – Making AI Actually Work for You

The workshop emphasized that **context is one of the most important factors** affecting the quality of AI-generated responses.

Key topics included:

- Why AI produces inaccurate answers when contextual information is insufficient.
- The concept of **Second AI Brain** for retaining and utilizing knowledge.
- Combining **Prompt Engineering** with contextual information to improve response quality.
- Designing AI systems capable of remembering and reasoning based on previous interactions.

#### Cloud Architecture with Amazon Bedrock

A complete serverless AI architecture was introduced using AWS services.

Main services included:

- Amazon CloudFront
- Amazon S3
- Amazon API Gateway
- AWS Lambda
- Amazon Bedrock
- Amazon DynamoDB
- AWS SAM
- React Application

The architecture demonstrated how users send requests through CloudFront and API Gateway, Lambda processes business logic, Amazon Bedrock performs AI inference, while S3 and DynamoDB store application data.

Benefits include:

- Fully serverless deployment
- Automatic scalability
- Reduced infrastructure management
- Lower operational costs
- Faster AI application development

#### Research on Large Language Models (LLMs)

The speakers introduced the research paper:

**"Non-Determinism of Deterministic LLM Settings"**

Main insights:

- The same prompt can still generate different responses.
- Various hidden factors influence LLM consistency.
- Understanding model behavior helps developers produce more reliable AI systems.

#### Understanding Temperature, Top-P and Top-K

The workshop explained how these parameters affect AI output.

- **Temperature** controls creativity and randomness.
- **Top-P** limits candidate words based on cumulative probability.
- **Top-K** limits the number of candidate tokens available for prediction.

Understanding these parameters enables developers to optimize AI responses for different business scenarios.

#### Single-Agent vs Multi-Agent Architecture

One of the most interesting sessions compared traditional Single-Agent systems with Multi-Agent Architecture.

Limitations of Single-Agent systems:

- Limited context
- Single point of failure
- Poor scalability
- Limited reasoning capability

The **Virtual Credit Committee** architecture demonstrated multiple specialized AI agents working together:

- Manager
- Financial Analyst
- Market Analyst
- Team Evaluator
- Risk Assessor
- Compliance Agent

This collaborative approach significantly improves reasoning quality, accuracy, maintainability, and scalability.
